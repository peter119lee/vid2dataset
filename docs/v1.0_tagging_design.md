# v1.0 Design — Auto-tagging + trainer-ready export

Route A (bounded integration), approved 2026-07-11. Port source:
`L:\Antigravitiy code\sd-image-sorter\backend\tagger.py` (1,688 lines → ~400),
plus lessons from that repo's tagger audit (`claude-code-sd-image-sorter-tagger-audit-REPORT.md`).

## Boundary rules (the anti-bloat contract)

1. `tagger.py` + `tagger_runtime.py` are a post-pipeline pass. Extraction core has
   ZERO imports from them; deleting both modules must leave extraction working.
2. Default OFF. Model (~1.2 GB) + onnxruntime (~40 MB) download on first enable only.
   The .exe does not grow.
3. Out-of-scope forever (write into README): training, upscaling, image editing,
   prompt tools. Dataset preparation only.

## Verified facts (2026-07-11)

- onnxruntime-directml 1.24.4 on PyPI, cp312 win_amd64 wheel exists, requires_python >=3.11.
- wd-eva02-large-tagger-v3 model.onnx = 1,260,435,999 bytes (~1.17 GB) on HF.
- hf-mirror.com serves the same paths (range probe 206). Endpoint order: huggingface.co → hf-mirror.com.
- WD v3 models: input NHWC 448x448, BGR, white letterbox pad, output = per-tag sigmoid probs,
  selected_tags.csv categories: 0=general, 3=copyright, 4=character, 9=rating.

## Modules

### `tagger.py` (new, ~400 lines)
- `TAGGER_MODELS`: `wd-eva02-large-tagger-v3` (default, per user), `wd-swinv2-tagger-v3` (light alt).
  Both same format — no per-model branching except repo_id + approx size.
- Download: urllib (reuse gpu_runtime `_download_with_progress` pattern: .part + rename),
  endpoints HF→hf-mirror, model cache `%LOCALAPPDATA%/vid2dataset/tagger_models/<name>/`,
  validation: size >1MB; corrupt (INVALID_PROTOBUF) → delete + one re-download (ported behavior).
- Preprocess EXACT port of the fixed version (audit F11): alpha-composite onto white,
  BICUBIC letterbox, white pad, RGB→BGR, NHWC float32.
- Inference: ONNX session, providers `["DmlExecutionProvider", "CPUExecutionProvider"]`
  (skip CUDA EP — DML covers NVIDIA and needs no cuDNN), batch with halving backoff on failure,
  GPU→CPU session fallback (simplified port), session recreate every 200 imgs on GPU (VRAM leak guard).
- Session injectable for tests (FakeSession).
- Caption writer (audit F2/F3/F6 lessons):
  - single line, `", "` joined, UTF-8, `newline="\n"` (LF, no CRLF)
  - order: trigger word (verbatim) → character tags (conf desc) → general tags (conf desc)
  - underscore→space EXCEPT kaomoji preserve list
    (`0_0 (o)_(o) +_+ +_- ._. <o>_<o> <|>_<|> =_= >_< 3_3 6_9 >_o @_@ ^_^ o_o u_u x_x |_| ||_||`)
  - parentheses kept literal (matches kohya's reference wd14 script; escaping
    would corrupt kaomoji, and training-side caption readers don't parse
    prompt weighting)
  - case-insensitive dedup keep-first; rating NEVER goes into captions (audit F1/F8)
  - cancelled runs mark unprocessed images `error="cancelled"`; tag_folder
    never writes sidecars for them (a trigger word would otherwise stamp
    fake captions onto images the model never saw)
- Public API: `tag_folder(folder, cfg, progress_cb=None, cancel_event=None) -> TagSummary`
  (walks images, tags, writes sidecars, returns per-tag frequency + counts).

### `tagger_runtime.py` (new, ~150 lines)
- Availability: `import onnxruntime` works → use it (source installs / dev venv).
  Else if frozen + cached → sys.path append activate. Else download.
- On-demand: `onnxruntime-directml==1.24.4` wheel via PyPI JSON (reuse gpu_runtime helpers
  `_query_pypi_url`, `_download_with_progress`, `_wheel_target`), extract to
  `%LOCALAPPDATA%/vid2dataset/tagger_runtime/`, manifest {version}, stale wipe like gpu_runtime.
- Non-Windows: no auto-download; instruct `pip install vid2dataset[tag]`.

### Config (`config.py` additions)
`tag_images: bool = False`, `trigger_word: str = ""`,
`tagger_model: str = "wd-eva02-large-tagger-v3"`,
`tag_general_threshold: float = 0.35 (0.05-0.95)`, `tag_character_threshold: float = 0.85`,
`kohya_repeats: int = 0` (0=off; >0 with flatten_output → images+txt in `output/{N}_{trigger}/`).

### Pipeline hook (`extractor.py`)
Order: extraction → (if cfg.tag_images) tagging pass → report/gallery/contact sheet,
so report+gallery can show tags. One call site, ~10 lines, lazy import.
Per-image failures logged, never fatal. Tag counts into `_run_summary.json`.

### CLI (`cli.py`)
`vid2dataset tag FOLDER --trigger X [--model M --threshold F --character-threshold F]`
— standalone tagging for any existing image folder.

### GUI (`app.py`)
Checkbox "Auto-tag images (WD tagger)" + trigger word entry + model dropdown (eva02/swinv2).
First enable with missing runtime/model → confirm dialog (~1.3 GB for eva02 / ~0.5 GB swinv2)
→ background download with per-file progress (same pattern as GPU runtime) → untick on failure.

### Report (`report.py`)
New section: top-30 tag frequency table + tagged/untagged counts.
Gallery hover: append tags line when sidecar exists.
